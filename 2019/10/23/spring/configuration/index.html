Configuration注解的使用

发表于 2019-10-23 | 分类于 Spring

不配置@Configuration

当内部方法bean发生彼此依赖的时候会导致多例，如下图所示

当加载UserService bean 的时候，如果没有配置@Configuration的话，会调用两次new MyService生成两个实例

配置@Configuration

当使用@Configuration的时候运行的结果如下图所示

可以看到当使用@Configuration注解的时候，两次调用返回的是同一个实例。
这是当使用@Configuration的时候，经过debug调试可以看到@Configuration将配置的类代理成了一个cglib代理对象，每次通过getBean的方式从容器中进行获取，而容器中的对象是单例的，所以返回的实例对象是同一个。
